# Work

1、首先是一份远程工作找到我了
房地产预订（区块链支持版）是一款先进的全栈网页应用，利用区块链技术扩展了传统的房地产预订平台，该系统允许用户浏览房源列表、保存收藏、预约房产参观，同时利用区块链不可篡改地记录预订、所有权验证和交易证明等关键作。
该应用结合了现代 React（Vite）前端、Node.js/Express 后端、MongoDB 数据库和区块链层（智能合约），打造了 Web2 + Web3 混合架构。Auth0 继续管理用户认证和身份，而区块链组件则引入去中心化信任、防篡改记录和可审计的预订记录。
该项目展示了区块链如何务实地集成到现有的全栈系统中 —— 作为安全关键作的补充层。

项目概述
房地产预订（区块链支持版） 是一个采用 Web2 + Web3 混合架构的全栈网页应用
核心功能
房源管理：用户可浏览房源列表、保存收藏、预约房产参观。
区块链赋能：利用区块链不可篡改的特性，记录预订、所有权验证和交易证明等关键操作，确保记录可审计、防篡改。
用户身份管理：通过 Auth0 实现安全的用户认证与身份管理。
技术栈
表格

架构设计亮点
该项目并非用区块链完全替代传统基础设施，而是将其作为安全关键操作的补充层，务实集成到现有全栈系统中，既保留了 Web2 应用的易用性，又获得了 Web3 技术带来的信任与安全优势。
技术架构解析
这是一个典型的 Web2 + Web3 混合架构 房地产预订系统，技术栈选择非常现代化，兼顾了开发效率、用户体验和区块链的信任特性。

前端（Web2 + Web3）
UI 框架：React + Vite，提供快速的开发体验和高性能的用户界面。
设计系统：Mantine，用于构建一致、美观的组件库。
路由管理：React Router，实现单页应用的页面导航。
网络请求：Axios，用于与后端 REST API 交互。
身份验证：Auth0 SDK，提供安全、便捷的用户登录和身份管理。
Web3 集成：
通过 ethers.js 或 web3.js 集成 MetaMask 等钱包，实现用户与区块链的交互。
提供交易状态反馈和确认，提升用户对区块链操作的感知。
后端
API 服务：Node.js + Express，构建 RESTful API 接口。
安全验证：Auth0 JWT，保护敏感路由，确保只有已认证用户才能访问。
业务逻辑：在调用区块链之前进行业务逻辑验证，避免无效或恶意的链上操作。
区块链交互：使用 ethers.js 作为中间层，封装与智能合约的复杂交互。
数据持久化：MongoDB 用于存储链下数据（如房源信息、用户资料）。
ORM 工具：Prisma 和 Mongoose，提供类型安全的数据库操作和模式管理。
智能合约层
合约语言：基于 Solidity 编写，这是以太坊生态最成熟的智能合约开发语言。
架构设计：采用事件驱动的后端同步设计，通过监听链上事件来更新链下数据库，保证数据一致性。
部署环境：部署到测试网（如 Goerli、Sepolia）或本地区块链（如 Hardhat Network），便于开发和测试。
架构亮点
混合架构：将 Web2 的易用性与 Web3 的信任机制相结合，既保证了用户体验，又利用区块链解决了房产交易中的信任和确权问题。
分层清晰：前端、后端、智能合约层职责明确，便于团队协作和后期维护。
安全可靠：通过 Auth0 进行身份验证，在区块链调用前进行业务逻辑校验，有效降低了安全风险。
